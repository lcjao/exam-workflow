# 📚 交互式学习库

> 基于 AI 的交互式考试备考系统 —— 文件即状态，命令即驱动

---

## 快速开始

### 第一次使用

1. 复制 `模板/` 目录下的模板文件到你的课程目录
2. 修改 `学习指南.md` 中的课程信息
3. 对 AI 说：**「使用 exam-workflow skill，帮我建立 [科目名称] 的交互备考系统」**
4. AI 自动生成全套学习文件

### 日常学习流程

```
1. 打开对应模块文件（Obsidian）
2. 阅读内容，在 > 答： 后填写答案
3. 对 AI 说「我答完了」
4. AI 评估 → 通过则推进，不通过则补强
5. 重复直到毕业
```

## 核心概念

### 文件即状态

所有学习状态都写入本地 Markdown 文件，AI 通过读取文件获取上下文。
换设备、换会话、换 AI 模型，都能恢复进度。

### 极简命令

| 命令 | 作用 |
|------|------|
| 「我答完了」 | 提交答案，触发评估 |
| 「我学完了」 | 完成阅读，触发评估 |
| 「补答完了」 | 完成补强，触发二次评估 |
| 「开始模块XX」 | 跳到指定模块 |
| 「我要毕业」 | 请求毕业评估 |
| 「复习」 | 手动触发复习 |

### 量化评估

每题按评分标准量化打分，≥80% 推进，<80% 补强。
详见 `模板/模块学习卡片.md` 中的评分规范。

### 间隔复习

基于掌握度自动生成复习计划，对抗遗忘曲线。
