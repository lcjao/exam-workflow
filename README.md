# Exam Workflow — AI 交互考试工作流

> 用 AI 把任何考试资料变成私人定制备考流水线。

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
[![Version](https://img.shields.io/badge/version-2.0.0-green.svg)](package.json)

---

## 一句话介绍

**Exam Workflow = 任何考试资料 × AI = 私人定制备考流水线。**

输入考试大纲或复习材料，AI 自动生成全套学习文件。学习过程中只需说「我答完了」，AI 自动评估、补强、推进，直到毕业。

---

## 核心特性

| 特性 | 说明 |
|------|------|
| 📁 文件即状态 | 学习记录写入本地 Markdown，跨会话/跨设备恢复进度 |
| 📊 量化评估 | 按题型权重逐题评分，≥80% 推进，<80% 自动补强 |
| 🔄 间隔复习 | 基于掌握度自动生成复习计划，对抗遗忘曲线 |
| 🎓 毕业机制 | 毕业评估 + 毕业笔记，明确知道什么时候算学完了 |
| ⌨️ 极简命令 | 5 个核心命令驱动全流程 |
| 🔓 完全开源 | MIT 协议，可自由修改和分发 |

## 快速开始

### 方式一：作为 AI Agent Skill 安装

将 `skill/` 目录复制到你的 AI Agent 的 skills 目录：

```bash
# Claude Code / Hermes Agent
cp -r skill/ ~/.claude/skills/exam-workflow/

# 或手动复制到对应目录
```

然后对 AI 说：

```
使用 exam-workflow skill，帮我建立 [科目名称] 的交互备考系统
```

### 方式二：Obsidian 用户直接使用

1. 复制 `learning-lib/模板/` 到你的 Obsidian vault
2. 修改 `学习指南.md` 中的课程信息
3. 开始学习，对 AI 说「我答完了」触发评估

### 方式三：npx 一键安装

```bash
npx @lcjao/exam-workflow
```

## 工作流概览

```
用户提供大纲/材料
       │
       ▼
  AI 生成学习文件
  ┌──────────────────────────┐
  │  学习指南.md              │
  │  模块01.md … 模块N.md     │
  │  进度追踪.md              │
  └─────────┬────────────────┘
            │
            ▼
  用户学习循环
  ┌──────────────────────────────────┐
  │  阅读模块 → 填写答案              │
  │       │                          │
  │  「我答完了」                     │
  │       │                          │
  │  AI 量化评估                     │
  │       │                          │
  │  ≥80% → 进入下一模块             │
  │  <80% → 薄弱点补讲 → 再评估      │
  │                                  │
  │  全部完成 → 「我要毕业」         │
  │       │                          │
  │  毕业评估 → 毕业笔记 → 归档      │
  └──────────────────────────────────┘
```

## 命令速查

| 命令 | 作用 |
|------|------|
| 「我答完了」 | 提交答案，触发 AI 评估 |
| 「我学完了」 | 完成阅读，触发评估 |
| 「补答完了」 | 完成补强练习，二次评估 |
| 「开始模块XX」 | 跳到指定模块 |
| 「我要毕业」 | 请求毕业评估 |

## 技术架构

- **知识结构化引擎**：LLM + 模板系统，自动拆解大纲
- **智能评估引擎**：量化评分 + 题型权重 + 薄弱点识别
- **文件状态管理器**：本地 Markdown 文件系统，零服务器依赖
- **间隔复习调度**：基于掌握度的自适应复习计划
