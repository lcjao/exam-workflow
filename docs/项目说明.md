## 📌 项目说明

本项目将多个科目的 AI 交互式备考经验，提炼为一套**通用 exam-workflow Skill**，安装后可对任意科目创建完整的 AI 辅助备考系统。

### 核心价值
- 用户只需提供**考试大纲/复习材料**，AI 自动生成全套学习文件
- 学习过程中用**简单命令**驱动 AI（如「我答完了」），无需复杂提示词
- 所有文件存储在 Obsidian 本地，**永久保存**，随时可查

## ✅ 设计模式提炼

从三个科目的实际使用中，提炼出以下核心设计模式：

### 模式1：线性模块推进 + 80% 门槛制
```
模块01 → [我答完了] → AI评估
  ├─ ≥80% → 进入模块02
  └─ <80% → 薄弱点补讲 → [补答完了] → 再评估 → 模块02
```

### 模式2：文件角色分工
- **进度追踪.md** = 项目大脑（记录全局状态）
- **模块XX.md** = 学习材料（知识输入 + 答题区）
- **专项练习.md** = 题型专攻（针对弱项的集中训练）
- **薄弱点补讲.md** = 错题本（专门记录和纠正典型错误）
- **总复习/冲刺.md** = 考前冲刺（高频考点压缩）
- **模拟试卷.md** = 最终检验（含答案，可自测评分）

### 模式3：Obsidian 文件即答题纸
用户将答案写在 `> 答：` 的引用块中，AI 读取文件后直接批改评分

### 模式4：渐进式生成
不一次性生成所有文件，随学习进展按需生成，保持灵活性

## 📝 使用说明

### 如何为新科目启动工作流

1. 在新对话中说：**「使用 exam-workflow skill，帮我建立 [科目名称] 的交互备考系统」**
2. 提供考试大纲或复习材料路径
3. 告知考试题型和分值分布
4. AI 自动生成全套文件

### 日常学习流程

1. 打开对应模块文件（Obsidian 或记事本）
2. 阅读内容，在 `> 答：` 后填写答案
3. 对话框说 **「我答完了」** + 文件路径（或 AI 已知当前文件）
4. 根据评估结果决定下一步

### 常见问题

**Q：AI 找不到文件怎么办？**
A：在命令后附上完整路径，如「我答完了，文件在 D:\...\模块01.md」

**Q：掌握度总是不到80%怎么办？**
A：第二次补强练习后，AI 会无论分数都推进，避免卡关

**Q：可以跳过某个模块吗？**
A：可以，直接说「开始模块XX」跳到任意模块
