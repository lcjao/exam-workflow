# AI 交互考试工作流 · 命令速查卡

## 核心触发命令（用户只需记这3个）

| 命令 | 触发时机 | AI 响应行为 |
|------|---------|------------|
| **「我答完了」** | 完成模块提问区 + 自测题 / 专项练习 / 模拟试卷 | 读取文件逐题评分，输出掌握度，列薄弱点 |
| **「我学完了」** | 完成阅读 + 练习（适合文章式学习） | 评估掌握度，≥80% 生成下一篇，<80% 给补强 |
| **「补答完了」** | 完成 AI 出的补强练习后 | 二次评估，≥80% 进入下一模块 |

---

## 进阶命令（按需使用）

| 命令 | 触发时机 | AI 响应行为 |
|------|---------|------------|
| 「开始模块01」 / 「开始模块XX」 | 准备学习指定模块 | 生成或打开对应模块卡片 |
| 「开始学习」 | 第一次启动学习 | 打开进度追踪，从当前状态继续 |
| 「做专项练习」 | 想做题型专项 | 询问题型后生成专项练习文件 |
| 「出模拟卷」 | 想做整卷模拟 | 生成完整模拟试卷并写入文件 |
| 「总复习」 | 所有模块完成后 | 生成或打开总复习冲刺文件 |
| 「薄弱点」 | 想针对弱点补强 | 分析历史掌握度，生成薄弱点补讲 |
| 「出速查」 | 考前快速复习 | 生成表格化核心考点速查手册 |
| 「出背一背」 | 需要记忆专项 | 生成适合背诵的精华提炼文件 |
| 「看进度」 | 了解当前学习状态 | 读取进度追踪文件，汇报当前状态 |
| 「继续」 | 上次中断后继续 | 读取进度追踪，从上次断点推进 |

---

## 评估返回格式（标准）

```
📊 本次评估结果

总体掌握度：XX%

✅ 掌握较好：
- [知识点]：[简评]

⚠️ 需要强化：
- [知识点]：[问题] → [改进方向]

📌 下一步：
[根据掌握度给出具体指引]
```

---

## 掌握度等级与含义

| 掌握度 | 状态 | 下一步 |
|--------|------|--------|
| ≥90% | 🏆 优秀 | 直接进入下一模块 |
| 80-89% | ✅ 良好 | 进入下一模块 |
| 70-79% | ⚠️ 中等 | AI 给出补强练习，说「补答完了」 |
| <70% | 🔄 需强化 | AI 重新讲解薄弱点，再次自测 |

---

## 门槛规则

- **≥80%** → 推进到下一模块/步骤
- **<80%** → AI 给出补强练习，用户完成后说「补答完了」触发二次评估
- 二次评估后无论分数，推进到下一步（避免无限循环）
