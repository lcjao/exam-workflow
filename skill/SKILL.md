# Exam Workflow

> 让每个备考者都拥有一个永不遗忘的私人 AI 教练 —— 通过 Obsidian 文件系统，让 AI 真正"记住"你的学习进度。

## When to Use This Skill

Use this skill when the user:

- 想系统化备考某科目（考研 / 期末 / 考证 / 职业资格）
- 已有学习资料（教材 / 大纲 / 笔记）希望 AI 拆解为模块
- 想把零散的 AI 对话转化为有状态、可追踪的备考流程
- 提到 "复习"、"模考"、"错题"、"冲刺"、"押题"、"薄弱点"
- 已搭建或愿意搭建 Obsidian 知识库
- 希望评估自己当前掌握度（量化打分）

**不适合**：

- 临时性的"问个题"型问答
- 用户明确只要"参考答案"（不想要学习过程管理）
- 没有任何学习材料、希望纯靠 AI 生成全部内容的科目（可作为冷启动起点）

## Core Philosophy（设计哲学）

| 原则 | 含义 | 反例 |
|------|------|------|
| **文件即状态** | 所有学习数据存到本地 Markdown，不靠对话记忆 | 每次问 AI "上次学到哪" |
| **命令驱动** | 5+6 个字面命令触发 AI 行为 | 每次写长 prompt |
| **量化评估** | 80% 门槛制 + 补强循环 | "感觉掌握得差不多了" |
| **权限分区** | AI/用户/共享三区，避免误改 | AI 误删用户答案 |
| **间隔复习** | 抗遗忘曲线，掌握度越低复习越频繁 | 学完就忘 |
| **渐进式生成** | 用到什么生成什么，节省 token | 一次性生成 9 个文件 |

## Commands（5 核心 + 6 辅助）

### 5 核心命令（字面值不可改）

| 命令 | 触发动作 | 使用场景 |
|------|----------|----------|
| **「我答完了」** | AI 读取答案 → 量化评估 → 更新进度 | 提交当前模块答案 |
| **「我学完了」** | AI 评估掌握度 → 决定推进/补强 | 完成阅读但未答题 |
| **「补答完了」** | AI 二次评估 → 无论如何推进 | 完成薄弱点补强练习 |
| **「开始模块XX」** | 生成指定模块的学习卡片 | 跳级或重学 |
| **「我要毕业」** | 生成毕业评估 → 判定是否达标 | 请求最终检验 |

### 6 辅助命令

| 命令 | 触发动作 |
|------|----------|
| **「跳过这题」** | 标记跳过，不纳入评分 |
| **「这个我会了」** | 标记知识点已掌握，跳过相关题 |
| **「出综合题」** | 生成跨模块综合练习 |
| **「申诉」** | AI 重新评估指定题目 |
| **「复习」** | 按复习调度出复习题 |
| **「继续」** | 从上次断点恢复（跨会话） |

## File Types（5 核心 + 2 按需）

### 核心文件（最小可用集，3 个就能跑通）

| 类型 | 文件名 | 职责 |
|------|--------|------|
| 课题地图 | `README.md` | 总览全局，模块列表 + 状态 |
| 模块学习卡片 | `模块01.md` … `模块N.md` | 知识输入 + 答题区 |
| 进度追踪 | `📊学习进度追踪.md` | 项目大脑，记录一切状态 |

### 按需解锁文件

| 文件名 | 触发条件 |
|--------|----------|
| `薄弱点补讲_XXX.md` | 评估 < 80% |
| `📅复习调度.md` | 完成 2+ 模块后 |
| `总复习_高频考点冲刺.md` | 所有模块完成 |
| `🎓毕业评估.md` | 用户说「我要毕业」 |

## Evaluation Specs（量化评估规范）

### 评分公式

```
模块掌握度 = Σ(题目得分 × 题目权重) / Σ(题目满分 × 题目权重) × 100%
```

### 题型权重

| 题型 | 单题满分 | 权重 | 说明 |
|------|----------|------|------|
| 判断题 | 10 分 | 0.5 | 基础认知 |
| 选择题 | 10 分 | 0.7 | 理解程度 |
| 简答题 | 20 分 | 1.0 | 核心知识点 |
| 分析题 | 30 分 | 1.5 | 应用能力 |
| 论述题 | 40 分 | 2.0 | 综合能力 |

### 等级阈值

| 等级 | 范围 | 行动 |
|------|------|------|
| ✅ 掌握 | 80-100% | 推进下一模块 |
| ⚠️ 模糊 | 60-79% | 生成薄弱点补讲 → 二次评估 |
| ❌ 未掌握 | 0-59% | 生成薄弱点补讲 → 二次评估 → 无论如何推进 |

### 评估输出模板

```
📊 模块XX 评估结果

总体掌握度：XX%（等级：✅/⚠️/❌）

逐题评分：
  Q1 [简答题] 16/20 权重1.0 → 得分率80%
    ✅ 核心概念正确
    ⚠️ 缺少具体数值支撑

  Q2 [分析题] 18/30 权重1.5 → 得分率60%
    ❌ 分析方向偏差
    📌 改进：重新阅读模块03

加权总分：XX分 / 满分XX分

📌 下一步：[具体指引]
```

## File Permission Zones（文件权限分区）

```markdown
<!-- AI_READ_ONLY -->
# AI 不得修改
- 用户填写的答案（> 答： 后面的内容）
- 用户写的自由区内容
<!-- END_AI_READ_ONLY -->

<!-- AI_WRITABLE -->
# AI 可以写入
- 评估结果
- 掌握度数据
- 进度状态
<!-- END_AI_WRITABLE -->

<!-- USER_ONLY -->
# 只有用户能改
- 学习笔记
- 个人感悟
<!-- END_USER_ONLY -->
```

**铁律**：

- AI 永远不修改 `AI_READ_ONLY` 区
- 用户永远不修改 `AI_WRITABLE` 区中的公式与阈值
- 进度数据由 AI 自动更新，用户可手动修正

## State Machine（状态机）

```
用户安装 Skill
       │
       ▼
 初始化（生成 README + 进度追踪）
       │
       ▼
[循环] 模块 N 学习
       │
       ├── 「开始模块XX」 → 生成模块XX.md
       │                         │
       │                    用户填写 > 答：
       │                         │
       ├── 「我答完了」→ 评估（见 4.1 公式）
       │                         │
       │                    ≥80% ──→ 进入下一模块
       │                    <80% ──→ 生成薄弱点补讲
       │                                  │
       │                            「补答完了」→ 二次评估
       │                                  │
       │                            推进（无论如何）
       │
       └── 完成所有模块 ──→ 生成总复习
                                  │
                            「我要毕业」→ 毕业评估
                                       │
                                 通过 → 毕业笔记
                                 未通过 → 列出薄弱模块
```

## Quick Start（5 分钟上手）

**Step 1**：确认你有一个 Obsidian vault（如果还没有，下载安装后新建一个 vault）

**Step 2**：在你的 vault 根目录创建 1 个新文件夹作为本次学习项目（如 `考研-数据结构/`）

**Step 3**：复制本 Skill 的所有模板到该文件夹

**Step 4**：向 AI 说：

> 我要备考 [科目]，请帮我用 Exam Workflow 初始化。

AI 将生成：
- `README.md`（项目地图）
- `📊学习进度追踪.md`（进度大脑）

**Step 5**：开始学习

> 开始模块01：[第一章节主题]

AI 生成 `模块01.md`，你在 Obsidian 中用 `> 答：` 格式答题。

**Step 6**：完成后告诉 AI

> 我答完了

AI 自动评估、推进或补强。
