# AI 交互考试工作流 Skill

## 🎯 Skill 用途

将用户的考试大纲、课程资料或复习材料，转化为一套完整的**本地 Obsidian 交互学习系统**，包含：

- 课题地图与进度追踪文件
- 分模块交互学习卡片（含提问区 + 自测题）
- 针对弱点的专项练习与薄弱点补讲
- 总复习冲刺文件
- 模拟试卷（含答案）
- 速查/背诵手册

用户学习过程中只需输入简单命令（如「我答完了」），AI 即自动读取文件内容、评估掌握度、推进下一步。

---

## 📂 文件系统架构

每个考试科目对应一个文件夹，包含以下标准文件（按学习顺序排列）：

```
科目名称_交互式学习/
├── README.md              ← 课题地图 + 工作流说明 + 掌握度记录
├── 📊学习进度追踪.md       ← 全局进度仪表盘（模块状态 + 知识点清单）
├── 模块01_XXX.md           ← 学习卡片（知识讲解 + 提问区 + 自测题）
├── 模块02_XXX.md
│   ...
├── 专项练习_分析题.md      ← 按题型专项训练
├── 专项练习_论述题.md
├── 薄弱点补讲_XXX.md       ← 错题/薄弱点二次讲解
├── 总复习_高频考点冲刺.md  ← 考前速查 + 高频问答
├── 📋核心考点速查.md       ← 精华速查手册（可选）
├── 00-背一背.md            ← 记忆/背诵专项（可选）
└── 模拟试卷_第X套.md       ← 完整模拟卷（含参考答案）
```

## 🚀 完整工作流程

### Step 1：接收用户输入

用户启动此 Skill 时，请主动向用户索取以下信息：

1. **考试大纲或复习材料**（文件路径 / 粘贴内容 / Obsidian 笔记路径）
2. **考试题型与分值分布**（如：选择20+改错30+简答20+分析15+论述15）
3. **目标文件夹路径**（默认在用户 Obsidian 库的考试目录下）
4. **模块划分**（如按章节、按知识模块，让用户确认或 AI 提议）

> 如果用户已提供材料，直接进入 Step 2，无需重复询问。

### Step 2：生成工作流文件

按以下顺序创建文件：

#### 2.1 创建 README.md（课题地图）

- 列出所有模块的学习状态表格（篇序 / 主题 / 对应考点 / 状态）
- 说明工作流步骤（4-5步，含触发命令）
- 提供掌握度记录表

#### 2.2 创建 📊学习进度追踪.md

- 整体进度（已完成X/N模块，当前状态，总体掌握度）
- 各模块学习状态表（含状态标记 ⏳📖✅🔄）
- 知识点清单（分模块，每条知识点含题型+状态格）
- 高频错误类型追踪（改错题等特定题型）
- 学习流程说明 + 触发命令提示

#### 2.3 创建模块学习卡片（模块01.md … 模块N.md）

每个模块文件遵循统一结构（详见 `references/module-template.md`）：

```
# 模块XX：主题 ⭐⭐⭐（重要程度）
> 重要程度 / 考试题型 / 学习目标

## 一、核心概念讲解
  - 知识点表格（含记忆技巧/口诀）
  - 理解逻辑（"为什么"推导链）

## 二、提问区 ✍️
  - 基础概念题 Q1-Q5
  - 深入理解题 Q6-Q8
  - > 答：（用户填写区）
  - 末尾：「回答完后告诉我"我答完了"」

## 三、自测题 🎯
  - 单选题 / 判断题 / 论述题

## 四、学习小结 ✅
  - 能力清单（checkbox 格式）
  - 🎯 下一步操作（含触发命令）
```

#### 2.4 创建专项练习文件

- 按题型创建（分析题/论述题/改错题/概念辨析等）
- 每题包含：题干 + 答题区（`> 答：`）
- 末尾固定：`答完告诉我"我答完了" → 逐题评分`

#### 2.5 创建总复习/冲刺文件

- 分板块汇总高频考点（含必背公式/判据/数值）
- 包含高频问答题（遮答自测）
- 末尾：`说"我答完了"触发评估`

#### 2.6 创建模拟试卷

- 按考试题型结构出完整试卷（满分100分）
- 用户填写在 `> 答：` 区
- 末尾：`答完后告诉我"我答完了" → 逐题评分`
- 第一套无答案，第二套附参考答案

### Step 3：响应触发命令（核心交互逻辑）

读取当前打开/指定的文件内容后，根据用户命令执行对应操作：

| 用户命令 | AI 响应行为 |
|---------|------------|
| 「我答完了」 | 读取文件，逐题评分，输出掌握度（0-100%），列出薄弱点 |
| 「我学完了」 | 评估掌握度，≥80% 生成下一篇，<80% 给出补强练习 |
| 「补答完了」 | 二次评估，更新掌握度，若≥80% 推进到下一模块 |
| 「开始模块XX」 | 生成对应模块学习卡片（若未生成），或提示打开已有文件 |
| 「做专项练习」 | 生成或打开对应题型的专项练习文件 |
| 「出模拟卷」 | 生成一套完整模拟试卷 |
| 「总复习」 | 生成总复习冲刺文件，或读取现有文件继续 |
| 「薄弱点」 | 根据历史掌握度记录，生成针对性薄弱点补讲文件 |
| 「出速查」 | 生成核心考点速查手册（表格+口诀形式） |
| 「出背一背」 | 生成记忆速背专项文件 |

### Step 4：评估时的标准输出格式

评估掌握度时，输出包含：

```
📊 本次评估结果

总体掌握度：XX%

✅ 掌握较好：
- 知识点A：[简评]
- 知识点B：[简评]

⚠️ 需要强化：
- 知识点C：[具体问题] → [改进方向]
- 知识点D：[具体问题] → [改进方向]

📌 下一步：
- 掌握度≥80%：[提示进入下一步]
- 掌握度<80%：[给出补强练习，答完说「补答完了」]
```

## 📐 设计原则

1. **Obsidian 文件即答题纸**：用户在 `> 答：` 区直接填写，AI 读取文件内容评估
2. **80% 门槛制**：掌握度≥80% 推进，<80% 补强后重测
3. **触发命令极简**：用户只需记住 `我答完了` / `我学完了` / `补答完了`
4. **文件角色分工清晰**：每类文件职责单一，不混用
5. **渐进式生成**：不一次性生成所有文件，随学习进展按需生成
